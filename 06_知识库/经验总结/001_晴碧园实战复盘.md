# 经验总结 — 晴碧园晶园26栋实战复盘

> 从真实项目DXF中学到的CAD制图经验与教训

## 一、图层命名

### 发现的问题
原始图纸用了大量**中文图层名**（W-尺寸、A-土建墙、P-活动家具等），这在AutoCAD中能用，但在：
- DXF解析时容易编码出错
- 跨软件（如LibreCAD）兼容性差
- 图块块名和图层名之间存在混淆

### 建议做法
```
原始:  A-土建墙（含管井、立管）
推荐:  A-WALL

原始:  P-活动家具
推荐:  P-FURN

原始:  W-尺寸
推荐:  A-DIMS
```

### 图层标准做法对照
| 原始图层 | 标准图层 | 问题 |
|---------|---------|------|
| W-尺寸 | A-DIMS | 图层分组不明确 |
| W-文字 | A-TEXT | W开头的含义不明 |
| P-完成面 | A-FNSH | 应在A组非P组 |
| W-墙体 | A-WALL | 同上 |

## 二、尺寸标注

### 发现
1. **118个 RotatedDimension** 全部在 W-尺寸 图层
2. 使用了3种标注样式：HG-60/HG-70/HG-80，对应不同比例
3. **95%+ 的标注没有颜色覆盖**（code_62=256），这是好习惯
4. 但113/118的图层不是标准A-DIMS

### 尺寸链规律
- 水平标注链（0°）：74条，集中在6组Y坐标
- 垂直标注链（90°）：44条，集中在22组X坐标
- 尺寸间隔标准值：200mm（墙厚/门垛）最常见（43次）

### 经验
✅ **颜色用ByLayer**（256） = 好习惯，方便统一改色
✅ **标注样式区分比例** = 好习惯，1:50/1:100用不同样式
⚠️ **图层名需标准化** = 原始图色彩混乱，但ByLayer策略补救

## 三、墙体绘制

### 发现
1. 790条LINE墙段，**438条 A-土建墙 + 352条 A-新隔墙**
2. 墙段平均长度 **1.1m** — 说明每段墙都是独立LINE非连续多段线
3. 纯LINE墙段**不形成闭合多边形**（门洞处断开）
4. 存在29个LWPOLYLINE墙（可能是楼梯间/电梯井）

### 墙体图层
- A-土建墙（含管井、立管）：438条，主结构墙
- A-新隔墙：352条，轻质隔墙
- 区分"土建"和"新隔" — **很好的习惯**，方便后期改造识别

### 经验
✅ **土建墙 vs 新隔墙分开** = 室内改造必备
⚠️ **LINE绘图不够闭合** = 对于面积计算，最好用LWPOLYLINE或HATCH

## 四、文字标注

### 发现
1. 104个MTEXT，全部用「建筑」样式
2. 高度统一：房间名 h=140，家具 h=100
3. 房间名简洁：卧室、客厅、卫生间
4. 家具标注详细：含品牌（如"烤箱""油烟机"）

### 经验
✅ **统一文字样式** = 好，避免了字体混乱
✅ **高度分层** = 140/100区分主次，但建议用整数倍数（3.5/2.5）
⚠️ 标注在W-文字层而非A-TEXT

## 五、图块管理

### 发现
1. **291个BLOCK定义 + 91个INSERT引用** 
2. 块名大多数是**匿名转换产物**（无意义字母组合）
3. 标准库块 YLD.Studio-CAD 出现7次
4. 最复杂的块 ZHOUXIAN（详图索引）含286实体

### 经验
✅ 使用外部块库（YLD.Studio）标准化家具/洁具
⚠️ 块名缺少标准化命名规范

## 六、HATCH填充

### 发现
1. 70个HATCH在原始图（晴碧园）
2. 366个HATCH在拆砌墙图
3. 9种图案类型，SOLID + _USER为主
4. **A-新隔墙填充**和**A-土建墙填充**图层分离

### 图案用法
| 图案 | 用途 |
|------|------|
| SOLID | 现浇混凝土填充、管井实心填充 |
| _USER | 新建隔墙斜线填充 |
| AR-CONC | 混凝土剖面 |
| 大理石3 | 地面材料示意 |
| AR-RROOF | 屋顶/露台示意 |

### 经验
✅ 新隔墙 vs 土建墙填充分开 = 一目了然
⚠️ 自定义图案_USER导致跨软件兼容问题

## 七、面积计算难点

### 为什么自动面积计算不准？

```
原始图纸: 
  790条LINE墙段（断开）
    ↓
门洞处断开 → 不闭合 → 无法直接求面积
    ↓
需要: 墙端桥接（补门洞）+ 窗洞修正
    ↓
V4方案: 桥接1500mm范围内墙端 → 32个闭合面
    ↓
局限: 桥接可能误连 → 大房间合并
```

### 最佳实践建议
1. **源头控制：** 画图时用LWPOLYLINE画房间轮廓
2. **HATCH法：** 用HATCH填充区域做面积基准
3. **手动法：** 在CAD中用AREA命令逐个房间量

## 八、图纸分析技巧清单

### 用OpenClaw分析DXF的通用流程
```
1. 批量解析 → 训练_批量解析.py
2. 图元分类 → 按type分组统计
3. 图层审查 → 颜色/线宽合规性
4. DIMENSION还原 → 尺寸链聚类
5. 文字提取 → 房间名+说明
6. INSERT图块 → 图块标准化
7. HATCH解析 → 填充类型识别
8. 综合报告 → 课题010
9. 轴网重建 → 课题012
10. 面积估算 → 面积统计工具
```

### 遇到过的坑
| 坑 | 表现 | 解决方案 |
|----|------|---------|
| 编码问题 | DXF中文变乱码 | 统一用UTF-8解析 |
| 浮点精度 | 墙端差0.001mm不闭合 | 四舍五入到1mm |
| 组码缺失 | 解析JSON少了HATCH边界 | 需升级DXF解析器 |
| 块名重复 | 匿名块无法区分 | 按块内容指纹哈希 |
| 坐标穿越 | 墙段跨整个楼层 | 按楼层Y阈值分割 |
