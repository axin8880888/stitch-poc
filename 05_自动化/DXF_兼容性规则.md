# DXF 兼容性核心规则（基于实战经验）

## ✅ 必须遵守

### 编码
- 使用 `ANSI_1252`（拉丁编码），不要用 `gb2312`（中文编码）
- 部分 Android CAD 看图软件不支持 gb2312

### 版本号
- `AC1015`（AutoCAD 2000）兼容性最好
- `AC1027`（2013）也可以
- 改版本号不会导致文件损坏

### 图层名替换
- 纯文本替换是安全的
- 等长替换最好（字节数不变）
- 不等长替换要**重新计算文件大小且不会破坏结构**

### 关于标准图层
- 可以逐行插入到 LAYER 表的正确位置
- 关键是：插在 `Defpoints` 图层定义结束之后、`ENDTAB` 之前
- 不能插在 `0 LAYER 2 0` 结束后的空行位置

## ❌ 会破坏文件的操作

### 过滤实体
- 跳过非 LINE 的实体需要正确处理 `0` 组码值
  - `60` 组码的值也可能是 `0`（不是实体结束标志）
  - 判断依据：`0` 后面紧跟的是否是类型名（`LINE`, `CIRCLE` 等）
- 跳过时可能同时跳过了重要的子类标记（`100/AcDbLine`）

### 错误插入位置
- `ENDTAB` 在 DXF 中多次出现（LTYPE/LAYER/STYLE/VIEW/UCS等表）
- 插入前必须确认是 LAYER 表的 `ENDTAB`（前面是图层定义的结束行）

## ✅ 正确的导出流程

```
原始 DWG → 转 DXF
  ↓
改版本 AC1027→AC1015
  ↓
改编码 gb2312→ANSI_1252  
  ↓
改图层名 A-新隔墙→A-WALL
  ↓
在LAYER表插入标准图层
  ↓
（可选）过滤非LINE实体
  → 必须确认0组码值判断正确
  ↓
验证：结构(SEC=END) + 编码 + 版本 + 图层
  ↓
PASS → 交付
```
