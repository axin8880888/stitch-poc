# 状态回写时序说明

## 目标
让接片原型的页面刷新与 JNI 处理节奏对齐。

## 时序

### 1. 初始化后
- `initNative()` 完成
- 预览页仍显示旧状态或暂无结果

### 2. 送帧中
- `pushFrameNative()` 每帧只影响内部 native 会话
- UI 不强制刷新

### 3. 拼接完成
- `stitchNative()` 返回后
- 写入：
  - `status = stitched`
  - `qualityScore`
  - `warnings`

### 4. 纯裁切完成
- `applyPureCropNative()` 返回后
- 写入：
  - `cropStatus = applied`
  - `cropBoundsSummary`

### 5. 导出完成
- `writeResultNative()` 返回后
- 写入：
  - `exportStatus = done`
  - `notes += 导出完成`

## 页面刷新
- 每次 `PocResultStore.save()` 后自动推送到预览页订阅者
- 预览页立即更新字段展示

## 原则
- 状态回写要快，但不能抢过 native 处理本身
- UI 只展示完成态，不做抢跑动画
- 所有关键节点都要可见、可检查
